## Recording Tips
- **Duration**: 2-3 minutes maximum
- **Tool**: Windows Game Bar (Win+G), OBS Studio, or ShareX
- **Resolution**: 1920x1080 recommended
- **Show**: Mouse cursor and clicks

## Script Timeline

### Introduction (15 seconds)
**What to show:**
- Open the application at localhost:3000
- Show the beautiful gradient UI

**What to say:**
> "This is Repository Mirror - an AI-powered GitHub analysis tool that evaluates any public repository and provides a score, summary, and personalized improvement roadmap."

---

### Demo 1: High-Quality Repository (45 seconds)
**URL to paste:** `https://github.com/facebook/react`

**What to do:**
1. Paste the URL
2. Click "Analyze"
3. Wait for results (2-5 seconds)
4. Scroll through showing:
   - High score (likely 80-95)
   - Score breakdown by category
   - Summary highlighting strengths
   - Key metrics (commits, files, stars)
   - Roadmap with positive feedback

**What to say:**
> "Let's analyze the React repository by Facebook. As you can see, it scores very high with excellent documentation, clean code structure, comprehensive tests, and consistent Git practices. The roadmap suggests advanced improvements."

---

### Demo 2: Average Repository (45 seconds)
**URL to paste:** `https://github.com/torvalds/linux`
*OR use any medium-quality project*

**What to do:**
1. Click back or refresh
2. Paste new URL
3. Click "Analyze"
4. Show the results:
   - Medium score (50-75)
   - Mixed feedback in summary
   - More actionable roadmap items

**What to say:**
> "Now let's try a different repository. This one has a more moderate score with room for improvement in areas like documentation and testing. Notice how the roadmap provides specific, actionable steps."

---

### Features Highlight (30 seconds)
**What to show:**
- Scroll back to show different sections
- Point out:
  - Real-time GitHub API integration
  - Score breakdown (6 categories)
  - Visual metrics dashboard
  - Emoji-enhanced roadmap
  - Responsive design

**What to say:**
> "The system analyzes 6 key dimensions: documentation quality, project structure, code quality, commit history, version control practices, and activity. Each gets weighted scoring for a total out of 100."

---

### Closing (15 seconds)
**What to show:**
- Quick scroll showing the full interface
- Maybe show the README in VS Code

**What to say:**
> "Repository Mirror helps developers understand their GitHub profile through data-driven insights and provides a clear path to improvement. Built with Python Flask, React, and real GitHub API integration. Thank you!"

## Recording Checklist

Before you start recording:
- [ ] Backend running (Flask on port 5000)
- [ ] Frontend running (Vite on port 3000)
- [ ] Browser open to localhost:3000
- [ ] Close unnecessary tabs/windows
- [ ] Turn off notifications
- [ ] Test one analysis to ensure it works
- [ ] Have your script ready
- [ ] Check audio levels if recording voice

During recording:
- [ ] Speak clearly and confidently
- [ ] Move mouse smoothly
- [ ] Wait for animations to complete
- [ ] Show enthusiasm about features
- [ ] Keep it under 3 minutes

After recording:
- [ ] Review the video
- [ ] Check audio quality
- [ ] Trim any dead time
- [ ] Export in MP4 format
- [ ] File size under 100MB recommended
